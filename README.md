## Babel

双语翻译浏览器扩展，支持网页全文翻译和 YouTube 字幕双语翻译。提供 25+ 翻译引擎、自定义规则、术语表、触屏手势、划词翻译和输入框翻译。

### 翻译引擎

**传统翻译** — Google · Google2 · Microsoft · AzureAI · DeepL · DeepLFree · DeepLX · Baidu · Tencent · Volcengine

**AI 大模型** — OpenAI · Gemini · Gemini2 · Claude · DeepSeek · SiliconFlow · Cerebras · AliyunBailian · XiaomiMimo · Zai · CloudflareAI · Ollama · OpenRouter · ePhoneAI · BuiltinAI · Custom

每个引擎可独立配置模型、Prompt、流式输出、批处理、占位符、请求频率和超时。

### 网页翻译

自动识别页面正文，支持全文翻译和鼠标悬停翻译两种模式。全文模式下，滚动到可视区域的段落自动翻译，离开视口后释放资源。

**核心能力**：

- 自动扫描或 CSS 选择器精确匹配翻译目标
- 富文本保留——HTML 标签、链接、术语标记原样保留，仅翻译文本内容
- 占位符系统——将复杂标签替换为占位符发送给翻译引擎，避免被破坏
- 流式渲染——AI 大模型逐字输出时，通过 requestAnimationFrame 锁帧渲染，丝滑不卡顿
- 视口锚点——插入译文后自动补偿滚动偏移，页面不抖动
- 段落切分——按标点或字符数自动切分长段落，避免超出 token 限制
- 长文本换行——超长原文翻译后自动换行，保持译文的可读性
- 语言检测——自动跳过与目标语言相同的文本，减少无效请求
- Shadow DOM——完整支持 Web Components 和 Shadow Root 内的文本翻译
- iframe 翻译——支持跨域 iframe 内文本翻译，可配置白名单

**交互方式**：

- 悬浮球 (FAB) —— 一键开关翻译、切换仅译文模式
- 鼠标悬停 + 快捷键 —— 鼠标指向段落按快捷键翻译，松开恢复
- 全局快捷键 —— 翻译开关、仅译文、样式切换、弹出菜单
- 触屏手势 —— 双指 / 三指点击、单指双击 / 三击开关翻译

### 字幕翻译

在 YouTube 等视频平台自动抓取字幕轨道，生成双语字幕叠加显示。

**断句与翻译**：

- AI 智能断句——将 YouTube 自动字幕的碎片化文本合并为完整句子后翻译
- 规则断句——基于标点和时间间隔的快速断句，无需额外 API 调用
- 上下文感知——携带前后句字幕作为参考上下文，消除断句翻译的突兀感
- 视频摘要——可选 AI 预分析视频内容生成术语表，提升专业内容的翻译准确度

**字幕交互**：

- 双语显示——原文和译文同时叠加在视频上，位置、字体、颜色、背景可自定义
- 模糊模式——译文模糊显示，用于听力训练
- 悬停查词——鼠标悬停在字幕单词上弹出词典释义和例句
- 生词本——点击单词加入生词本，支持导出
- 字幕列表——侧边栏全文滚动显示字幕历史，点击跳转视频进度
- 拖拽定位——字幕窗口可自由拖拽和调整位置

**字幕增强**：

- 日语 ASR 字幕结束时间自动延长到下一句起始
- 译文在上、原文在下的双语排序
- 广告字幕自动跳过

### 划词翻译

选中页面任意文本弹出翻译面板。支持必应词典和有道词典查词，输入联想补全，以及生词收藏。

### 输入框翻译

在任意网页输入框中输入文本，按快捷键即时翻译并替换。支持多语言互译。

### 规则系统

为每个网站配置独立的翻译规则——选择器、翻译引擎、源语言、目标语言、富文本模式、样式等。规则支持订阅，可自动从远程 URL 更新。

**全局规则**：对所有网站生效的默认翻译行为。**子规则**：按 URL 匹配网站，规则支持 CSS 选择器前缀（`+` 追加、`-` 排除）。

### 安全

Trusted Types CSP 完整支持。页面安全策略拒绝 Trusted Types Policy 时，自动降级为 DOM Fragment 注入，确保翻译内容在任何安全策略下都能正常渲染。所有 HTML 注入统一经 DOMPurify 消毒。

### 上下文控制

4 个独立开关按场景拆分，可按需开启以平衡翻译质量和 API 消耗。

**API 设置**中可开关：

- 页面元数据上下文——将当前页面的标题、描述注入翻译 Prompt，帮助术语一致性
- 多轮对话历史——将前几句翻译作为上下文发送，维持术语统一。可配置保留轮数

**字幕设置**中可开关：

- 字幕前后句上下文——携带相邻字幕文本作为参考，解决碎片化翻译的突兀感
- AI 视频摘要——翻译前先调用 AI 分析视频内容。提升专业内容准确度，但额外消耗 API

AI 视频摘要默认关闭。页面元数据和字幕前后文默认开启。

### 同步

支持 WebDAV 和自建 Worker 两种同步方式，在多台设备间同步设置、规则、生词本。

### 自定义

- 自定义 Prompt 模板——为每个 AI 引擎编写系统提示和用户提示，使用 `{{title}}`、`{{tone}}` 等占位符
- 自定义样式——为译文编写 CSS 样式，支持模糊、高亮、字体、颜色等
- 术语表——配置术语和对应译文，翻译时强制使用
- AI 术语——JSON 格式的领域词典，注入 Prompt 供 AI 参考
- 翻译钩子——翻译前后的 JS 脚本钩子，可修改文本和请求参数
- 自定义 CSS / JS 注入——在目标页面注入额外样式和逻辑

### 构建

需要 Node.js 18+ 和 pnpm。

安装依赖：`pnpm install`

构建 Chrome：`pnpm build:chrome`，输出在 `build/chrome/`

构建 Firefox：`pnpm build:firefox`，输出在 `build/firefox/`

### 安装

打开 `chrome://extensions/`，开启开发者模式，加载 `build/chrome/` 目录。

### YouTube 字幕批量翻译策略

YouTube 字幕翻译采用 chunk 批量请求，不再按单条字幕逐句请求。当前默认切分为 `20 / 40 / 80`：先独立翻译首个 20 条字幕 chunk，首屏译文可用后再并发翻译剩余 chunks。

核心行为：

- 首屏优先：chunk0 成功后即可启动双语字幕显示。
- 后续全并发：chunk0 成功后，剩余 chunks 同时发起翻译请求。
- 严格 JSON：AI 必须返回 `{"items":[{"id":0,"text":"..."}]}`，解析器校验数量、id、重复项和空译文。
- 上下文辅助：每个 chunk 携带前后少量字幕上下文，context 使用 `globalIndex`，不会混入待翻译 `id`。
- 失败重试：chunk 解析失败或缺译时最多重试 3 次，避免乱序或空译文进入显示层。

DeepSeek `deepseek-v4-flash` benchmark 参考数据，1464 条字幕：

```txt
20 / 40 / 80, first chunk first then all remaining chunks concurrently
first chunk: ~2.9s
total: ~13.1s
failed chunks: 0
```

成本粗估：1464 条字幕约 20 个请求，DeepSeek Flash 约几分钱到一两毛人民币级别，主要成本来自输出 tokens。
